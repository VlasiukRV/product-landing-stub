Technical Appendix

System Architecture

The system is built around a real-time audio ingestion pipeline designed for low-latency, fault-tolerant speech processing.

AudioShipper → WebSocketAudioStream → Dispatcher → Output Clients
/01

AudioShipper

Asynchronous client captures PCM frames and streams them via WebSocket with minimal jitter.

/02

WebSocketAudioStream

A channel-bound buffer that normalizes incoming signals for the transcription engine.

/03

Dispatcher Layer

Guarantees strict channel isolation, routing audio to independent processing contexts.

/04

Client Layer

Real-Time live text & audio output
